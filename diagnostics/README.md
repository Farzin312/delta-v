# Diagnostics: Readiness Assessment

> Course labels are too coarse. "I took Calculus II" says little about
> convergence, series, or integration fluency. Every module begins with
> a capability contract and a diagnostic.

This directory contains self-administered diagnostics for each knowledge
spine module. They identify your weak prerequisite nodes so you can repair
them before proceeding.

---

## How to Take a Diagnostic

1. No notes. No AI. No calculator (unless the diagnostic says otherwise).
2. Time yourself lightly (don't obsess, but don't dawdle).
3. Score honestly. A correct answer with a weak explanation is NOT mastery.
4. Use the readiness scoring below to decide your next action.

### Readiness Scoring

| Result | Action |
|--------|--------|
| 85-100% | Proceed to the module. Schedule cumulative review. |
| 65-84% | Proceed, but embed targeted repair into the week. |
| Below 65% | PAUSE. Repair the failed nodes first. |
| Correct answer, weak explanation | Treat as NOT mastered. Reconstruct the reasoning. |

## Repair Strategy

When a diagnostic reveals a weak node, repair ONLY that node. Do not
repeat an entire course.

### The Repair Process

```
1. Identify the specific failed node
   (e.g., "logarithm properties," not "all algebra")

2. Study the formal concept
   (read the relevant section, work through examples)

3. Solve 2-6 focused exercises
   (not 50 — just enough to confirm the node is repaired)

4. Apply it once in an integrated context
   (connect it back to what you're trying to learn)

5. Re-take the diagnostic question
   (can you now solve it without notes?)
```

### Example

Diagnostic M0 reveals you can't convert scientific notation reliably.
Don't re-take all of algebra. Do:
1. Read about scientific notation (Khan Academy, 15 min)
2. Work 4 conversion problems by hand
3. Use it in Session 01 (convert 12.5 ms = 12.5 x 10^-3 s)
4. Re-take the M0 question that failed
