# delta-v

The energy cost of changing trajectories.

Code implementation of the [Frontier Engineer Field Manual](https://github.com/Farzin312/delta-v) - a self-paced, dependency-driven curriculum for becoming a computational space systems engineer.

Three spines, built simultaneously:
- **Physical reasoning** - mechanics, dynamics, calculus, linear algebra, numerics, controls, estimation
- **Software systems** - Rust first; Python for science/AI; C/C++ interoperability; testing, OS, embedded, real-time
- **Evidence and judgment** - requirements, uncertainty, validation, independent references, failure analysis

Every equation becomes an executable. Every implementation gets tested, broken deliberately, and verified independently. No borrowed understanding.

## Structure

```
first_30/          # Launch sessions (highest detail, highest friction)
  practice_1/      # Units, vectors, kinematics - the first equation made executable
  practice_2/      # ...
```

## The loop

1. **Predict** - sign, scale, direction before any code
2. **Derive** - from first principles, with units and dimensions
3. **Implement** - one clean Rust function with typed interfaces
4. **Test** - known values, boundaries, properties, failure cases
5. **Falsify** - break it deliberately, understand why
6. **Explain** - ownership statement: what you trust, why, where it fails

Advance only when the mastery gate is met. No fixed calendar.
